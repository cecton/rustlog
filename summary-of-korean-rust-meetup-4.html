Summary of Korean Rust Meetup #4

Rustlog ⋅ 2015-01-20 11:17 +09:00

As I've promised before, this is a summary of the 4th Korean Rust Meetup hosted by Korean Rust User Group, where people concentrate on getting things done no matter they are working on (libraries, Rust PRs and so on).

@klutzy was repairing suruga's experimental X.509 parser. There was some work on the certificate verification for a while, but recent macro reforms broke them a lot. Taking this as an opportunity to fully develop the parser, the development branch of suruga now got at least one kind of extensions working.

@kroisse worked on the Rust port of libearth, a supporting library for the Earth Reader.

I (@lifthrasiir) added a new testing script for Rustdoc outputs. Actually I wasn't able to send a pull request in time (the PR was sent hours after the meetup), as I accidentally omitted NO_REBUILD=1 and wasted some hours. :S

@sanxiyn got a basic infrastructure for code completion. At the moment it is able to produce a set of possible field names at given file position. This will tremendously help the development of IDEs for Rust.

@simnalamburt worked on obj-rs which parses Wavefront .obj file. We already have some existing parsers, but I've been told that other parsers cannot handle files produced by proprietary softwares, such as 3ds Max, so their uses are somewhat limited. It will help many professional game developers I guess.

There were also some interested people who are simply willing to learn Rust. This is partly because the meetup was co-located with other meetups (such as PyJog). Having 1.0.0 alpha surely was good for them! I'm looking forward to the next meetup, which would likely be on the next month (February 2015).

The Rustlog is a weblog for Rust-related posts, written by Kang Seonghoon (aka lifthrasiir and Yurume). It used to be a daily worklog of whatever he has done with Rust every day but he got bored and it is now more like a yearly log (sorry!). See also archives and his main journal (mostly in Korean).
